# 小红书售卖：发货不及时的解决方案

## 问题

- 用户付款后需你**手动**发激活码和登录链接。
- 激活码每个不同，无法在小红书里设「固定内容」自动发货。
- 发货不及时会导致成交率下降、用户投诉。

---

## 方案对比

| 方案 | 是否自动 | 实现难度 | 说明 |
|------|----------|----------|------|
| **A. 预生成 + 卡密平台** | ✅ 全自动 | 低 | 用有赞/微店/小鹅通等，上传一批激活码，付款后自动发卡密 |
| **B. 自助领取页** | ⚡ 半自动 | 中 | 用户付款后到网站填订单号，你审核后点「发放」，用户自助查激活码 |
| **C. 提高手动速度** | ❌ 手动 | 低 | 模板消息、手机提醒、5 分钟内必发，降低流失 |

---

## 推荐组合：B + C（自助领取 + 快速响应）

1. **自助领取页**：用户付款后到固定链接填「订单号 + 手机后 4 位」→ 你在管理后台点「发放」→ 系统自动分配一个未使用的激活码 → 用户在同一页或查询页**自助查看**激活码和登录链接，无需你私信复制粘贴。
2. **快速响应**：有新待发放时手机提醒（可选），或养成每天固定时间处理；未发放前用户会等待，你在 5～15 分钟内处理即可明显提升体验。

这样：
- 你不用「复制激活码再私信」每个用户，减少遗漏和延迟。
- 用户有明确预期：「到链接里填信息，等几分钟后刷新就能看到码」。
- 成交率提升：流程清晰、等待时间可控。

---

## 方案 A：卡密自动发货（真·全自动）

若希望**完全自动**、零人工：

1. **预生成一批激活码**  
   ```bash
   docker exec -it nihongo-go node scripts/generate-activation-code.js 100 0
   ```
2. **导出为卡密列表**  
   从数据库或脚本输出导出为一行一个的 txt/csv。
3. **在支持「卡密自动发货」的平台卖**  
   - **有赞**、**微店**、**小鹅通** 等支持「卡密/兑换码」商品，上传这批码，用户付款后自动收到一条消息（含一个卡密）。
4. **商品说明**  
   写清：付款后自动收到激活码，并附登录/激活链接：`https://nihongo-tool.cn/activate`。

**注意**：用户拿到的是「卡密」，即你的激活码；用户需要再打开你的网站 `/activate` 输入该码完成激活。若平台支持「卡密+链接」一起发，可把链接写进卡密说明或自动发货模板。

## 方案 B：自助领取页（本站实现）

本站已提供：

1. **用户侧**  
   - 打开 **领取页**（如 `https://nihongo-tool.cn/claim`）。  
   - 填写：订单号（小红书订单号或你约定的单号）、手机号后 4 位（用于核验）。  
   - 提交后显示「已提交，请等待发放」。

2. **管理员侧**  
   - 打开 **管理后台 → 发货管理**（或 `/dashboard/admin/deliveries`）。  
   - 看到「待发放」列表，核对是否已付款（可对照小红书订单）。  
   - 点击「发放」：系统自动从池子里分配一个**未使用**的激活码给该订单。  
   - 状态变为「已发放」。

3. **用户再次打开领取页**  
   - 再次输入同一订单号 + 手机后 4 位。  
   - 若已发放，则直接展示：**激活码** + **登录/激活链接**（如 `https://nihongo-tool.cn/activate`）。  
   - 用户无需等你私信，减少遗漏和延迟。

这样你只需「审核 + 点发放」，不用逐个复制粘贴发私信，用户也能自助查码，提升成交率和满意度。

## 方案 C：提高手动发货速度（无开发）

不接任何新功能时，可先优化流程：

1. **固定话术模板**  
   - 例如：「您的激活码：FURIGANA-XXXX-XXXX，激活链接：https://nihongo-tool.cn/activate ，请先激活并设置密码后使用。」
2. **手机强提醒**  
   - 小红书若支持「新订单提醒」，务必打开；或设「每 10 分钟看一次订单」的闹钟。
3. **承诺并执行**  
   - 商品页写「付款后 5 分钟内发放」，并尽量在 5 分钟内发完，降低流失。

---

## 建议落地顺序

1. **短期**：上线 **自助领取页 + 管理后台发放**（方案 B），并在商品页/私信里说明「付款后请到该链接填写订单号与手机后 4 位，约 5～15 分钟内可在此页查看激活码」。  
2. **中期**：若单量上来，再考虑 **有赞/微店/小鹅通 + 预生成卡密**（方案 A），实现全自动发货。  
3. **始终**：保持 **快速响应**（方案 C），避免用户长时间等待。

---

## 小结

- **无法提前设置「统一自动发货」** 是因为每个激活码不同；要么用**卡密平台**预导入一批码实现自动发，要么用**自助领取页**把你从「复制粘贴私信」里解放出来，并让用户自助查码。  
- 推荐先做 **自助领取 + 管理后台发放**，再视情况接入卡密自动发货平台。
